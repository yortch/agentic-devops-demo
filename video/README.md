# Agentic DevOps — Video

A ~70-second animated video built with [Remotion](https://www.remotion.dev/) showcasing the Agentic DevOps demo — from chaos injection to autonomous incident recovery.

> **🤖 This entire video project — scaffold, screenshots, scene components, composition, and this README — was created using [GitHub Copilot CLI](https://githubnext.com/projects/copilot-cli).** No manual code was written. The project structure and Remotion skills were adapted from [danielmeppiel/apm-onemin-video](https://github.com/danielmeppiel/apm-onemin-video).

## Quick Start

```bash
# Install dependencies
npm install

# Open Remotion Studio (live preview)
npm run dev

# Render final video
npx remotion render AgenticDevOps out/agentic-devops.mp4
```

## How It Was Built

This video was **created entirely using GitHub Copilot CLI** with the Remotion framework. No code was written manually.

### The Prompt

The following prompt was given to GitHub Copilot CLI to build this video:

> I want to create an outstanding ~1 minute video for LinkedIn that goes primarily over the `docs/index.html`. Here's a rough script:
>
> **Step 1** — Capture images using dark mode:
> - Issue #56 for the selected chaos scenario
> - Workflow run showing the chaos workflow triggered manually, expanding the Agentic Conversation
> - PR #70 showing the code change introduced ready to merge
>
> **Step 2** — Screenshot of the CD deploy workflow run
>
> **Step 3** — Azure Monitor alert screenshot
>
> **Step 4** — SRE Agent investigation screenshot (provided as step-4.png)
>
> **Step 5** — Issue #71 created by SRE Agent with root cause analysis
>
> **Step 6** — PR #72 with the fix created by Copilot Coding Agent
>
> **Step 7** — PR #72 review comments
>
> **Step 8** — Production restored screenshot (provided as step-8.png)

### Build Process

1. **Scaffold** — Project structure and Remotion skills adapted from [danielmeppiel/apm-onemin-video](https://github.com/danielmeppiel/apm-onemin-video)
2. **Screenshots** — Dark-mode GitHub screenshots captured via Playwright browser automation (injecting `data-color-mode="dark"` on each page navigation)
3. **Scenes** — React components with spring animations, staggered entrances, and fade/slide transitions
4. **Pan-scroll** — Steps with long GitHub pages (issues, PRs) use a zoomed-in pan animation to improve readability
5. **Composition** — All scenes wired via `TransitionSeries` with 15-frame crossfades
6. **Iteration** — Duration compressed from ~95s to ~71s, font sizes increased, pan-scroll added — all through follow-up prompts

## Customization

- **Timing** — Adjust `*_DUR` constants in each scene component
- **Colors** — Edit `COLORS` in `src/styles/theme.ts`
- **Step content** — Edit the `STEPS` array in `src/styles/theme.ts`
- **Transitions** — Change `TRANSITION_FRAMES` in theme.ts (currently 15 frames = 0.5s)
